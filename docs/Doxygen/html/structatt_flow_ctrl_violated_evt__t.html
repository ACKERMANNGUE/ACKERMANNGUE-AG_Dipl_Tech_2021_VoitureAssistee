Detailed Description

ATT Flow Control Violated Event message format.

The following two ATT events are generated locally (not received OTA) by the ATT Server or Client.

This message is sent as a GATT_MSG_EVENT of method ATT_FLOW_CTRL_VIOLATED_EVENT to the app by the local ATT Server or Client when a sequential ATT Request-Response or Indication-Confirmation protocol flow control is violated for a connection. All subsequent ATT Requests and Indications received by the local ATT Server and Client respectively will be dropped.

This message is to inform the app (that has registered with GAP by calling GAP_RegisterForMsgs) in case it wants to drop the connection.
